  28.      * The Jf::sql function. The behavior of this function is as follows:
  29.      *
  30.      * * On queries with no parameters, it should use query function and fetch all results (no prepared statement)

  32.      *      bound to question marks.
  33.      * * On SELECT, it will return 2D array of results or NULL if no result.
  34.      * * On DELETE, UPDATE it returns affected rows
  35.      * * On INSERT, if auto-increment is available last insert id, otherwise affected rows
  36.      *
  37.      * @todo currently sqlite always returns sequence number for lastInsertId, so there's no way of knowing if insert worked instead of execute result. all instances of ==1 replaced with >=1 to check for insert
